## B. データ構造

### B.1 入力JSON（`content/input/<slug>.json`）

- [x] `title` / `description` / `category` / `turns[]` を持つ構造を受け付ける (§3.1) — `InputLesson` 型
- [x] 各 turn が `turn_id` / `speaker` / `voice_setting` / `text` / `translation` を持つ (§3.1) — `InputTurn` 型
- [x] `voice_setting` は kokoro-js 対応のボイス名を指定できる (§3.1) — voices/ から検証

### B.2 レッスンメタデータ（`content/lessons/<slug>.json`）

- [x] `id` / `slug`（ファイル名から導出）を付与する (§3.2)
- [x] `title` / `description` / `category` を保持する (§3.2)
- [x] `turns[]` に `turn_id` / `speaker` / `text` / `translation` / `audio_path` / `audio_hash` を含める (§3.2) — 生成済み JSON で確認
- [x] `audio_path` は相対パス形式（例: `/audio/at-the-cafe/1.mp3`）(§3.2)
- [x] `audio_hash` は `text` ＋ `voice_setting` から算出する (§3.2)

## C. CLI生成スクリプト（`npm run generate`）

### C.1 入力読み込み・スラッグ

- [x] `content/input/` 内の全 JSON を読み込む (§4.1)
- [x] スラッグはファイル名から拡張子（`.json`）を除いてそのまま使用（自動変換しない）(§4.1)

### C.2 入力バリデーション（生成前・エラー時は中断）

- [x] 問題があれば**どのファイル・どの行か**を示してエラー終了する (§4.1)
- [x] 不正なまま生成・出力しない (§4.1)
- [x] **話者3種類以上**でバリデーションエラー（2人想定）(§4.1)
- [x] 大文字小文字・前後空白のみ異なる話者名の混在をバリデーションエラー（暗黙の正規化はしない）(§4.1)

### C.3 音声生成

- [x] 各 turn の `text` ＋ `voice_setting` を kokoro-js で音声化する (§4.1)
- [x] **1ターンずつ逐次処理**する（並列化しない）(§4.1)
- [x] kokoro-js の出力（PCM/WAV）を MP3 へエンコードする (§4.1)
- [x] MP3 を `public/audio/<slug>/<turn_id>.mp3` に保存する (§4.1)
- [x] MP3 は低ビットレート（48〜64kbps / mono / 24kHz 程度）でエンコードする (§6) — 実測 64kbps/24kHz/mono

### C.4 メタデータ出力・差分再生成

- [x] `audio_path` と `audio_hash` を付与して `content/lessons/<slug>.json` に出力する (§4.1)
- [x] 既存メタデータの `audio_hash` と比較し再生成要否を判定する (§4.1)
- [x] ハッシュ一致の行はスキップ（再生成しない）(§4.1)
- [x] ハッシュ不一致／新規の行のみ再生成し `audio_path`・`audio_hash` を更新する (§4.1)
- [x] メタデータ欠落・音声ファイル欠落の行も生成対象とする (§4.1)
- [x] `--force` で全行強制再生成できる（任意）(§4.1)

### C.5 クリーンアップ

- [x] 入力から削除された行に対応する音声・メタデータを削除する (§4.1)
- [x] 入力JSONごと削除された場合、対応する `content/lessons/<slug>.json` と `public/audio/<slug>/` を自動削除する (§4.1)

### C.6 CLI体験・運用

- [x] ターン単位で進捗表示（例: `[1/5] turn 1 — generating...`）し、スキップ／生成／エラーを明示する (§4.1)
- [ ] 生成物（音声・メタデータ）を Git にコミットして履歴管理する (§4.1) — 運用時にユーザーがコミット
- [x] 生成はローカルのみ・認証不要（管理画面/APIを本番にデプロイしない）(§4.1)

## H. 音声プレイヤー（操作は「全体再生」＋「行タップ」の2つに集約）

- [x] 専用のリピートボタンを設けない (§4.2)
- [x] **全体再生**: ユーザー操作起点で最初から最後まで連続自動再生する (§4.2)
- [x] 各行を自動で繋いで連続再生する (§4.2) — `onEnded` で次行へ
- [x] **行タップ**: その行を開始位置に以降を最後まで連続再生する (§4.2)
- [x] 再生中の行をハイライト表示する (§4.2)
- [x] 行の進行に合わせて自動スクロールする (§4.2) — `scrollIntoView`
- [x] **同じ行の再タップで頭出し**（リピート兼用）できる (§4.2) — `playIndex` が常に先頭から
- [x] 音声は `public/audio/` 配下の静的ファイルを直接参照する（サーバー不要）(§4.2)
- [x] 次の行の音声を先読み（preload）して無音の隙間を最小化する (§7.7) — 先読み用 `<audio>`

### H.1 再生速度

- [x] 速度切替（`0.5x` / `0.75x` / `1.0x` / `1.25x`）を提供する (§4.2)
- [x] `playbackRate` を用い音程を大きく崩さず変更する (§4.2)
- [x] 速度は全体再生・行タップ再生に共通適用する (§4.2) — 単一 audio 要素に適用
- [x] 再生を止めずに速度変更でき即時反映する (§4.2) — `audio.playbackRate` を即時更新
- [x] 選択速度を `localStorage` に保存し再訪時も維持する (§4.2)

### H.2 キーボードショートカット

- [x] `Space` — 再生 / 一時停止トグル (§7.7)
- [x] `→` — 次のターンへ進む（最終ターンでは停止）(§7.7)
- [x] `←` — 現在再生中ターンの頭出し（先頭ターンでも頭出し）(§7.7)

### H.3 エラー・欠落時

- [x] 音声ファイルが取得できない行は**スキップ**して次へ進む (§6) — `onError` で次行へ
- [x] スキップ行に控えめなエラーインジケータ（例: 行番号の色変更）を表示する (§6) — 番号 line-through ＋ `NO AUDIO`

## K. UI/デザイン（Teenage Engineering トーン）

### K.1 原則

- [x] ミニマル: 余白を大胆にとり要素を詰め込まない (§7.1)
- [x] 機能主義: 装飾的グラデーション・強い影・過度な丸みを避ける (§7.1) — グラデ/影なし・角丸 4px
- [x] 厳格なグリッド・整列（8px グリッド等の一貫したリズム）(§7.1) — `--space-unit:8px`

### K.2 カラー

- [x] ベースはほぼモノクロ（オフホワイト／グレー／ブラック）(§7.2)
- [x] アクセントはオレンジ `#FF5500` に固定し、再生中行・主要アクション・現在状態のみに限定使用する (§7.2)
- [x] 多色使いをしない (§7.2) — エラー表示も赤を使わずグレー＋取り消し線
- [x] ライト基調を標準とし、同原則でダークモードを用意できる構成にする (§7.2) — `prefers-color-scheme`

### K.3 タイポグラフィ

- [x] 本文・UIラベルはサンセリフ (§7.3) — Geist Sans
- [x] 番号・話者名・ステータス・ボタンラベルはモノスペース (§7.3) — Geist Mono
- [x] ラベルは短く、必要に応じ大文字＋トラッキング（字間広め）(§7.3) — `.label-mono`
- [x] 英文（学習対象）は可読性優先で十分な行間・文字サイズを確保する (§7.3) — `text-base sm:text-lg leading-relaxed`

### K.4 コンポーネント

- [x] ボタンは角丸控えめ・フラットな矩形、線の細い幾何学アイコン (§7.4) — lucide アイコン
- [x] 表示モード切替はハードウェアスイッチ調の明快なトグル (§7.4) — 押下状態が明快な EN / EN+JA トグル
- [x] 会話行は吹き出し装飾を避け、話者ごとに整列・番号付けした端正なリスト/カード (§7.4)
- [x] 再生中の行はアクセントカラーで明確にハイライト (§7.4)
- [x] 再生位置・モード・進捗を計器的でシンプルなインジケータで示す (§7.4) — `NN / NN` のモノスペース表示

### K.5 実装方針（デザイントークン）

- [x] カラー・フォント・余白スケール・角丸・状態色を `globals.css` の CSS 変数（`:root`）で一元定義する (§7.5)
- [x] Tailwind テーマ・shadcn/ui が `globals.css` のトークンを参照する設定にする (§7.5) — `@theme inline`
- [x] 個別コンポーネントに色・サイズを直書きしない（調整は原則 `globals.css` のみ）(§7.5) — トークン由来ユーティリティのみ使用

## M. CSR インタラクション UX

- [x] 再生/一時停止・行タップ・モード切替をクライアント状態のみで即時反映する (§7.7)
- [x] レッスン一覧⇄個別ページを `next/link` のソフトナビゲーションで遷移する（フラッシュ回避）(§7.7)
- [x] 表示モード・再生速度などの設定を `localStorage` に保存する (§7.7)
- [x] モード切替・スクロール時も再生位置・状態・ハイライト行を途切れさせない (§7.7) — 表示系 state は audio に非干渉

- [x] 主要操作はキーボードでも行える（H.2 参照）(§7.7)
